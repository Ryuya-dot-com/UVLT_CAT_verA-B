# Security and protected research content

Do not report a vulnerability by attaching participant data, Prolific identifiers, API tokens, completion codes, HMAC keys, D1 exports, source workbooks, stimuli, or answer keys to a public issue.

The public repository is code-only. Any pull request that introduces `data/`, `cloudflare/private/`, row-level exports, secrets, licensed stimuli, canonical keys, correctness fields, scores, or IRT parameters must be rejected.

If a credential may have been exposed, revoke or rotate it first through the relevant service, preserve a restricted incident record, and contact the repository owner privately through an agreed institutional channel.
